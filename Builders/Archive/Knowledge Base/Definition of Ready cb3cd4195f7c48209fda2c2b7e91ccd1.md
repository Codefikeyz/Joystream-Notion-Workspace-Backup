# Definition of Ready

- Well-defined User story. The story should be written exactly in the ‘user story’ format.
- User story is properly created in Project 56, tagged with relevant tags.
- Stakeholders defined. Who is this user story built for?
- User story acceptance criteria defined and understood by the Builders team.
- User story sized by the JSG (assigned SP)
- Builders team accepts user experience artifacts
- Performance criteria identified, if applicable
- Builders team is able to ‘demo’ the user story
