# Bounty Process

[L1.Media](http://L1.Media) Role

- Provides Bounty tasks
- Acts as Oracle for bounty
- Assigns reward to each bounty
- Can assign Bounty to individual
- Decides the reward of the Bounty manager
- Can extend grace period to individuals to complete task
- Merges PR to Repo

Codefikeyz Role

- Onboarding of potential bounty workers
- Follow up on bounty tasks
- Assigns tasks to Participants
- Verifies eligibility of participants to execute the bounty (what are the criteria?)

        Criteria: Active participation, expertise, previous 
        experience with task, native speaker/fluency 
        (language tasks), Pioneer membership. Other 
        criterias can be added e.g following our social 
         media pages.

- Pushes request to commit changes to Repo
- Scouts for reviewers for bounty tasks
- Suggests reward for review to @l1dev  (final decision is up to him)
- Assists bounty takers with orientation of tasks
- Provides template for task (what is template?)

        Template: This can be (a sample) for transcript, 
        translation, design template. Example [here](https://www.canva.com/design/DAFmnLb2Uc4/30k_IbCY65E5pMlQqTDHCA/view?utm_content=DAFmnLb2Uc4&utm_campaign=designshare&utm_medium=link&utm_source=publishsharelink&mode=preview)

- Provides documentation, links and necessary info to execute task
- Announces task assignment to participant
- Announces completion of task
- Forwards participants data to [L1.media](http://L1.media) to pay rewards
